# Data Science Pipeline — Wine Quality (DVC + MLflow)

This directory contains a reproducible ML pipeline demonstrating **data versioning with DVC**, **pipeline orchestration**, and **experiment tracking with MLflow**. 

**Dataset**: [UCI Wine Quality (red)](https://archive.ics.uci.edu/dataset/186/wine+quality) — 1,599 samples predicting wine quality from chemical properties.

---

## 📁 Structure

```
data-science/
├── data/
│   ├── raw/winequality-red.csv.dvc   # DVC pointer (tracked in git)
│   └── prepared/                     # generated by pipeline, git-ignored
├── src/
│   ├── prepare.py                    # clean + split data
│   ├── train.py                      # train model, log to MLflow
│   └── evaluate.py                   # generate evaluation plots
├── dvc.yaml                          # pipeline stages
├── params.yaml                       # hyperparameters
├── .dvc/config                       # S3 remote configuration
└── requirements.txt
```

## ⚡ Quick Start (5 minutes)

### 1. Setup Environment

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Configure DVC Remote

Get the S3 bucket name from Terraform:
```bash
DVC_BUCKET=$(cd ../infrastructure && terraform output -raw dvc_bucket_name)
dvc remote modify s3remote url "s3://$DVC_BUCKET"
dvc config core.remote s3remote
```

Or run the auto-config script:
```bash
bash setup-dvc-s3.sh
```

### 3. Pull Data

```bash
dvc pull  # pulls data/raw/winequality-red.csv from S3
```

### 4. Run Pipeline

```bash
export MLFLOW_TRACKING_URI="http://localhost:5000"  # or your remote MLflow host
dvc repro
```

Pipeline runs in order:
1. **prepare** → cleans & splits data
2. **train** → trains RandomForest, logs to MLflow
3. **evaluate** → generates evaluation plots

## 📚 Complete Setup Guide

### Prerequisites

- AWS CLI configured: `aws sts get-caller-identity`
- Python 3.8+: `python3 --version`
- Git configured: `git config user.name`

### Infrastructure Setup (One-time)

```bash
cd ../infrastructure
terraform apply --auto-approve
terraform output dvc_bucket_name  # Note this
```

This creates:
- **S3 Bucket**: `{cluster-name}-dvc-store` for DVC storage
- **IAM Policy**: Least-privilege S3 access (ListBucket, GetObject, PutObject, DeleteObject)
- **IRSA Role**: For Kubernetes pods to access S3 without hardcoded credentials

### Local Development Setup

```bash
cd ../data-science
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# Configure DVC remote
DVC_BUCKET=$(cd ../infrastructure && terraform output -raw dvc_bucket_name)
dvc remote modify s3remote url "s3://$DVC_BUCKET"
dvc config core.remote s3remote

# Verify configuration
dvc remote list
cat .dvc/config
```

### Adding New Data Files

```bash
# Add a CSV file to DVC
dvc add data/raw/your-file.csv

# Commit metadata to git (NOT the data file itself)
git add data/raw/your-file.csv.dvc .gitignore
git commit -m "Add your-file.csv tracked with DVC"

# Push data to S3
dvc push

# Teammates can now pull it
dvc pull
```

## 🎯 Common Commands

### Data Management

```bash
# Add file/folder to DVC
dvc add data/raw/file.csv

# Download tracked files from remote
dvc pull

# Upload tracked files to remote
dvc push

# Check status of tracked files
dvc status

# See what changed
dvc status -c  # -c = check remote
```

### Pipeline Management

```bash
# Run entire pipeline
dvc repro

# Run specific stage
dvc repro prepare

# Force re-run all stages
dvc repro --force

# View pipeline DAG (dependencies)
dvc dag --ascii

# Validate pipeline
dvc dag --check
```

### Experiments & Hyperparameter Tuning

```bash
# Run with different hyperparameters
dvc exp run --set-param train.n_estimators=300
dvc exp run --set-param train.max_depth=10

# Show all experiments and metrics
dvc exp show

# Compare specific experiments
dvc exp show exp1 exp2

# Export results to CSV
dvc exp show --csv > results.csv

# Restore experiment as current branch
dvc exp apply <exp-id>

# Remove experiment
dvc exp remove <exp-id>
```

### Metrics & Results

```bash
# Show metrics from all experiments
dvc metrics show

# Show metrics with more decimals
dvc metrics show -M 4

# Compare metrics between commits
dvc metrics diff HEAD~1

# Plot metrics
dvc plots show

# Show specific metric plot
dvc plots show metrics.json:loss
```

### Configuration

```bash
# List all remotes
dvc remote list

# Add a new remote
dvc remote add myremote s3://bucket/path

# Set default remote
dvc remote default myremote

# View current config
cat .dvc/config

```

## 🔧 Running the Pipeline

### Local Development

```bash
cd data-science
source .venv/bin/activate

# Option 1: Local MLflow (SQLite backend)
mlflow ui --backend-store-uri sqlite:///mlflow.db &
export MLFLOW_TRACKING_URI="http://localhost:5000"

# Option 2: Remote MLflow (on K8s)
export MLFLOW_TRACKING_URI="http://<your-mlflow-ingressroute-host>"

# Run pipeline
dvc repro

# View experiment metrics
dvc exp show
dvc metrics show
```

### Experimenting with Hyperparameters

```bash
# Try different hyperparams without editing code
dvc exp run --set-param train.n_estimators=400
dvc exp run --set-param train.max_depth=12

# Compare all experiments locally
dvc exp show
```

Each `dvc exp run` creates a corresponding **MLflow run**, so you get:
- **DVC**: Lightweight local experiment comparison
- **MLflow**: Full run history, UI, model registry

## 🐳 Running in Kubernetes (Optional)

### IRSA Setup

Kubernetes pods need AWS credentials to access S3. This is handled via **IRSA** (IAM Roles for Service Accounts) — same pattern used for RDS IAM auth and Secrets Manager.

```bash
# Annotate ServiceAccount with IRSA role
kubectl apply -f ../k8s/dvc-runner-manifests.yaml

# Verify annotation
kubectl -n mlflow describe sa dvc-runner
# Should show: eks.amazonaws.com/role-arn: arn:aws:iam::...
```

### Run Pipeline in K8s

```bash
# Create and run a training job
kubectl apply -f ../k8s/dvc-pipeline-job.yaml

# Monitor job
kubectl -n mlflow get jobs
kubectl -n mlflow logs -f job/dvc-train-wine-quality

# View results in MLflow UI
```

Example K8s Job that runs `dvc repro`:
```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: dvc-pipeline
  namespace: mlflow
spec:
  template:
    spec:
      serviceAccountName: dvc-runner
      containers:
      - name: dvc
        image: python:3.11
        command: ["bash", "-c"]
        args:
          - |
            git clone https://github.com/your-org/mlflow.git /workspace
            cd /workspace/data-science
            pip install -r requirements.txt
            export MLFLOW_TRACKING_URI="http://mlflow-service.mlflow.svc.cluster.local:5000"
            dvc pull && dvc repro
      restartPolicy: Never
```

## ❓ Why Both DVC and MLflow?

| Tool | Purpose | In This Project |
|------|---------|-----------------|
| **DVC** | Version control for data + pipelines | Manages CSV files, tracks pipeline stages, enables reproducibility |
| **MLflow** | Experiment tracking + model registry | Logs params/metrics/models, provides UI, enables collaboration |

- **DVC** answers: "Which exact data + code + params produced this model?"
- **MLflow** answers: "What happened during this run? Where's the resulting model?"

Together they form the backbone of production ML: reproducible pipelines (`dvc repro`) logged to a central registry (MLflow) that can be versioned, compared, and promoted.

## 🔐 Security & Credentials

**No AWS access keys anywhere in this project.** `.dvc/config` carries no credentials:

- **Locally**: Uses default AWS credential chain (`aws configure` / SSO profile)
  ```bash
  aws sts get-caller-identity  # Verify your credentials work
  ```

- **In Kubernetes**: Uses **IRSA** (temporary credentials via OIDC)
  ```bash
  kubectl -n mlflow describe sa dvc-runner
  # Verify IRSA annotation exists
  ```

**S3 Security**:
- Least-privilege IAM policy (only needed S3 permissions)
- Public access blocked
- Versioning enabled for data recovery

## 🐛 Troubleshooting

### `dvc pull` fails with "No credentials found"

```bash
aws configure
aws sts get-caller-identity  # Verify access
dvc pull
```

### S3 remote URL is wrong

```bash
dvc remote list
dvc remote modify s3remote url "s3://your-correct-bucket"
dvc config core.remote s3remote
```

### Pipeline doesn't update after changing data

```bash
dvc status
dvc repro --force  # Force re-run all stages
```

### "Access Denied" in Kubernetes

```bash
# Verify IRSA role is attached to ServiceAccount
kubectl -n mlflow describe sa dvc-runner
# Should show: eks.amazonaws.com/role-arn: arn:aws:iam::...
```

### MLflow connection fails

```bash
# Verify tracking URI is set
echo $MLFLOW_TRACKING_URI

# Test connectivity
curl -I $MLFLOW_TRACKING_URI

# If local, start MLflow
mlflow ui --backend-store-uri sqlite:///mlflow.db &
```

## ✅ Best Practices

### ✅ DO

- Commit `.dvc` files to Git, not the data
- Use `params.yaml` for hyperparameters
- Declare all dependencies in `dvc.yaml`
- Run `dvc repro` locally before pushing
- Use `dvc exp run` for hyperparameter sweeps
- Push data after successful runs: `dvc repro && dvc push`

### ❌ DON'T

- Commit large data files (CSVs, models, pickles) to Git
- Hardcode AWS credentials in code or `.dvc/config`
- Mix DVC-tracked and Git-tracked versions of same file
- Push data without committing `.dvc` metadata
- Skip `.gitignore` entries for DVC-managed files
